# 全流程与最终目标

## 核心原则

这个仓库不是“只筛股”，而是做一条完整闭环：

`委托 -> 建任务 -> 筛股 -> 七层研究 -> 分析决策 -> 规则复盘 -> 结果落盘 -> 闭环`

## 1. 委托入口

你在飞书群里把事情交给 `小C`。

例子：

- 今天先给我一份候选池
- 研究一下山西汾酒能不能建仓
- 复盘一下今天为什么只推了 3 只票

## 2. 小C 建任务

系统把这条委托转成一个正式任务。

任务至少包含：

- `task_id`
- `intent`
- `owner`
- `status`
- `current_assignee`

## 3. 分流到不同意图

当前先支持三类：

- `daily_candidate_pool`
- `single_stock_research`
- `rule_update`

## 4. 每日候选池流程

```text
你 -> 小C
  -> 小巴筛候选池
  -> 小C追办落盘
  -> 关闭
```

输出：

- 当日候选池
- 候选原因
- 是否存在明显缺口

## 5. 单票七层研究流程

```text
你 -> 小C
  -> 小智补七层数据
  -> 小巴做分析
  -> 小C追问/收口/落盘
  -> 关闭
```

七层固定口径：

1. 行情
2. RSI
3. 财务
4. 资金面
5. 催化剂
6. 题材
7. 风险点

输出：

- 单票七层研究卡
- 建议：建仓 / 观望 / 放弃
- 原因与证据

## 6. 规则更新流程

```text
研究结果/复盘结果
  -> 小C识别是否有规则沉淀价值
  -> 生成 rule suggestion
  -> 已知规则强化可自动落盘
  -> 新规则等待你审批
```

输出：

- 强化已有规则
- 新规则建议草案
- 需要你决定的边界项

## 7. 闭环定义

你已经定下来的闭环标准是：

- 不是发一条消息就算结束
- 不是做了一半静默就算结束
- 任务要么落地
- 要么明确进入 `WAITING_USER`
- 要么明确告诉你卡在什么地方、缺什么审批

## 8. 最终成熟目标

最终要做成：

- 多 agent 协同透明
- 研究链路可追踪
- 复盘和经验可积累
- 模拟盘流程可持续
- 后续具备扩展到更高级自动化的基础

当前不追求一步到位。

先把以下三件事做成熟：

1. 每日候选池
2. 单票七层研究卡
3. 经验规则自动更新
