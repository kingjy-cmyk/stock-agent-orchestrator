# 中文产品介绍

## 这是什么

`stock-agent-orchestrator` 是一个给多 agent 选股协同体系提供“后台任务大脑”的仓库。

它不替代你现在的飞书群，也不要求你放弃 `小C / 小智 / 小巴` 这套分工。

它解决的是一个更根本的问题：

`你把一件事交给小C后，不能做着做着没下文。`

## 解决的问题

当前纯群聊协同的主要短板是：

- 任务靠上下文记忆，不靠系统状态
- 谁该接下一步，很多时候靠 bot 自己“记得”
- 缺证据、缺落盘、缺催办时，容易沉默掉线
- 同一个委托经常被拆成多轮对话，而不是一个持续跟进的任务

这个仓库要把上面的弱点改成下面的能力：

- 每个委托都有任务 ID
- 每个阶段都有责任人
- 每次推进都有状态变化
- 卡住时明确进入 `WAITING_USER`
- 已知规则内自动推进
- 新规则只生成建议，等你审批

## 目标产品形态

### 控制面

- `飞书群`

你仍然在群里和 `小C / 小智 / 小巴` 互动。

### 核心大脑

- `Task Engine`
- `Risk Engine`
- `Memory Engine`

这部分放在后台，负责持续跟进，不依赖单轮聊天记忆。

### 观察台

- 后续补 `Web Panel`

你可以随时看到任务卡在哪一步、谁没回复、哪里没落盘。

## 三个外显角色

- `小C`：任务 owner、闭环经理、沉淀负责人
- `小智`：筛股、拉七层数据、盘中监测
- `小巴`：分析、建仓建议、规则解释

完整运行前提详见：[运行前提与最小配置](PREREQUISITES_ZH.md)。

## 三个隐藏模块

- `Orchestrator`：任务编排和状态流转
- `Risk Engine`：规则判断和审批边界
- `Memory Engine`：经验总结、规则草案、复盘沉淀

## 当前仓库已经有的能力

- 任务状态机
- 审批边界
- 当前 `candidate_list.md` 桥接
- 当前七层研究报告桥接
- 规则建议生成
- 基础 CLI

## 最终目标

把仓库逐步做成一个成熟的、多 agent 协同的、可持续进化的模拟炒股研究系统。

阶段目标不是一步到位，而是：

1. 先把 `每日候选池` 做稳
2. 再把 `单票七层研究卡` 做稳
3. 再把 `规则自动总结` 做稳
4. 再接 `shadow mode`
5. 再接 `beta group`
6. 最后才考虑更深的自动化
